Preguntas a responder

  • ¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?

    • La validación en el lado del cliente permite proporcionar retroalimentación instantánea al usuario sobre los errores que pueda cometer al completar el formulario. Puede reducir la carga en el servidor ya que este no necesita procesar y validar datos potencialmente incorrectos, y puede proporcionar una capa adicional de seguridad al formulario, al evitar que se envíen datos maliciosos o incorrectos al servidor.

  • ¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?

    • La forma más simple de evitar las validaciones de JavaScript es desactivar JavaScript en el navegador. También puede que un usuario malintencionado podría intentar modificar el código JavaScript de la página para evitar la ejecución de las validaciones. En el peor de los casos es que el usuario tenga conocimeintos técnicos sabe cómo funcionan las validaciones del lado del cliente, podría intentar enviar datos directamente al servidor sin pasar por esas validaciones.

  • Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?

    • Si bien las validaciones en el lado del cliente con JavaScript son una buena práctica para mejorar la experiencia del usuario y reducir la carga del servidor, no deben considerarse como una medida de seguridad completa por sí solas. Es importante implementar validaciones en el lado del servidor para garantizar la seguridad y la integridad de los datos.
