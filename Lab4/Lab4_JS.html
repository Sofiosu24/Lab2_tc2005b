Laboratorio 4

  • ¿Qué diferencias y semejanzas hay entre Java y JavaScript?
    • Diferencias
    1. Interpretado vs. compilado: Java es un lenguaje compilado, el código fuente se compila en bytecode que se ejecuta en la máquina virtual Java (JVM). JavaScript, por otro lado, es un lenguaje interpretado que se ejecuta en un navegador web.
    2. Titularidad del lenguaje: El control de Java está en manos de Oracle. Javascript se basa en un estándar abierto y es gestionado por ECMAScript, una organización sin ánimo de lucro
    3. Tipado: Java es un lenguaje de programación de tipado estático, hay que indicar su tipo y este no podrá cambiarse a lo largo de la ejecución del programa. Por su parte, Javascript es un lenguaje de tipado dinámico que permite variar el tipo de dato de una variable las veces que se desee.
    Similitudes
    1. Sintaxis básica: A pesar de sus diferencias semánticas y de ejecución, Java y JavaScript comparten algunas similitudes en la sintaxis básica.
    2. Plataformas multiplataforma: Tanto Java como JavaScript son utilizados en múltiples plataformas. Java es conocido por su lema "write once, run anywhere" (escribe una vez, ejecuta en cualquier lugar), mientras que JavaScript es compatible con todos los principales navegadores web y puede ser ejecutado en diversas plataformas a través de Node.js.
  • ¿Qué métodos tiene el objeto Date? (Menciona al menos 5*)
    • Date.now(): Devuelve el valor númerico correspondiente al actual número de milisegundos transcurridos desde el 1 de Enero de 1970.
    • Date.parse(): Transforma la cadena que representa una fecha y retorna el número de milisegundos transcurridos desde el 1 de Enero de 1970.
    • Date.UTC(): Acepta los mismos parámetros de la forma extendida del constructor (por ejemplo: del 2 al 7 ) y retorna el número de milisegundos transcurridos desde el 1 de Enero de 1970.
    • Date.prototype.getDate(): Retorna el día del mes (1-31) para la fecha especificada acorde a la hora local.
    • Date.prototype.getDay(): Retorna el día de la semana (0-6) para la fecha especificada acorde a la hora local.
  • ¿Qué métodos tienen los arreglos? (Menciona al menos 5*)
    • push(): Añade un elemento al final del arreglo
    • pop(): Este elimina un elemento del final de un arreglo. Entonces este regresa con el elemento eliminado y cambia el arreglo original.
    • shift(): Eliminar un elemento desde el principio del arreglo y regresa el elemento eliminado y cambia el arreglo original.
    • slice(): Puede copiar y clonar un arreglo a un nuevo arreglo usando el metodo
    • Array.isArray(value): Regresa verdadero si el valor que pasa es un arreglo.
  • ¿Cómo se declara una variable con alcance local dentro de una función?

    • En JavaScript se puede declarar una variable con alcance local dentro de una función utilizando la palabra clave var, let, o const, dependiendo de si se desea que la variable sea mutable o inmutable, y el alcance deseado.

  • ¿Qué implicaciones tiene utilizar variables globales dentro de funciones?

    • El uso de variables globales dentro de funciones puede tener implicaciones problemáticas. Las variables globales pueden dificultar la comprensión del código, especialmente en proyectos grandes o complejos y si no se gestionan adecuadamente, las variables globales pueden tener nombres que coincidan accidentalmente con variables locales dentro de una función, lo que puede llevar a comportamientos inesperados y errores difíciles de depurar.

Laboratorio 5

Material Design es un conjunto de principios de diseño desarrollado por Google que se centra en la creación de interfaces de usuario consistentes, atractivas y funcionales para aplicaciones y sitios web. Fue anunciado por primera vez en 2014 y desde entonces ha sido ampliamente adoptado en numerosas plataformas y aplicaciones.

Los principios básicos de Material Design incluyen:

Material
Se inspira en la idea de que el diseño digital debe reflejar las propiedades físicas del mundo real.
Movimiento significativo
El movimiento se utiliza para proporcionar retroalimentación visual y mejorar la comprensión de las acciones del usuario.
Color y tipografía
Material Design hace un uso extensivo de colores vibrantes y tipografía clara y legible para mejorar la accesibilidad y la usabilidad.
Diseño adaptativo
Se adapta a diferentes dispositivos y tamaños de pantalla para garantizar una experiencia de usuario consistente en todas las plataformas.
Iconografía
Se proporciona un conjunto de iconos coherentes y escalables que pueden utilizarse para representar diferentes acciones y conceptos dentro de una aplicación o sitio web.
